# Подписка — План сборки

Единственный источник **стадии репозитория**. В начале каждой сессии Claude Code читает этот файл и определяет текущий этап — первый не-`DONE`.

Пять этапов. Больше нет, потому что игра — это два правила подписи и ни одного состояния.

**Ядро и фабрика в этом плане не участвуют.** Оба готовы в devnet-объёме (ядро S0–S3, фабрика F0–F5, `factories[]` пиннуты, `crown-salt` полон — соль `stream` уже там, с паритет-фаззом против задеплоенной программы) и не трогаются ни одной строкой.

Всё, что не в таблице, — в `post-game.md`. Не строим. Не готовим место. Не оставляем хуков.

## Принцип

Снизу вверх. Один этап = один связный проверяемый кусок. Не переходи дальше, пока DoD не выполнен и тесты зелёные. Меньше кода — меньше ошибок: у каждой строки есть текущий потребитель.

### G0 — Скелет: два крейта, границы, CI

**Выход.** Cargo-воркспейс: `logic/` и `canister/`. `config/testnet.toml`, `config/mainnet.toml` по схеме game-spec §9. `dfx.json`. CI: fmt, clippy, test, и шесть структурных линтов ниже.

**Структурные линты (это и есть суть этапа).**
1. `cargo tree -p subscription-logic --edges normal` печатает **только сам крейт**. Ноль зависимостей.
2. `grep -rE 'ic_cdk|std::(fs|net|time)|reqwest' logic/src/` → пусто.
3. `grep -riE 'solana|ed25519|sha256|candid' logic/src/` → пусто. Логика не знает про чейн и криптографию.
4. `grep -riE 'transfer|approve|splitter|treasury' canister/src/` → пусто; `grep -r 'http_request' canister/src/` → пусто. Канистра не двигает деньги и не читает внешние сети.
5. `grep -rE 'stable_structures|set_timer|StableBTreeMap' canister/src/` → пусто. Состояния и таймеров нет — это заявлено спекой и проверяется машиной.
6. Парсер `.did`: update-методы — ровно `{get_resolver, request_release, request_cancel}`. Любой сверх списка роняет CI.

**DoD.** Дерево совпадает с картой в `CLAUDE.md`. Шесть линтов зелёные на пустом репозитории. Ни одного сетевого значения, адреса или principal в коде.

### G1 — `logic` — расписание

**Вход.** game-spec §4.

**Выход.** Крейт `subscription-logic`. В нём: `due_at(t0, period, index) -> Option<i64>` (checked, зеркало ончейн-арифметики формы), `release_due(now, t0, period, index) -> Result<(), ScheduleError>` — единственный закон игры, `LOGIC_VERSION`. Время — аргумент. Ни адресов, ни хешей: расписание — чистые числа.

Заголовок `lib.rs`:
```rust
#![forbid(unsafe_code)]
#![deny(clippy::unwrap_used, clippy::expect_used, clippy::panic,
        clippy::arithmetic_side_effects, clippy::indexing_slicing)]
```

**DoD.** Property-тесты (`proptest`, dev-dependency) доказывают:
- **паритет арифметики** — `due_at` совпадает с ончейн `due_at` формы на фазз-наборах, включая отрицательные `t0` и переполнения (`None`, не паника);
- **закон** — `release_due` ⇔ `now ≥ t0 + index·period`; до срока — ошибка; переполнение срока — ошибка, не паника;
- **монотонность** — созревший кусок не перестаёт быть созревшим с ростом `now`; сроки кусков строго возрастают по `index` при `period > 0`;
- **детерминизм** — один вход даёт побитово один результат.

Плюс: `LOGIC_VERSION` определён и покрыт тестом на неизменность при пересборке.

### G2 — Канистра: деривация и выпуск

**Вход.** game-spec §2, §7, §9, §10.

**Выход.** Канистра поверх `logic`. Update-методы: `get_resolver`, `request_release`. Пересборка `salt` формы `stream` через `crown-salt` (resolver — из деривации `[subscription_id]`, не из аргументов), адрес через `crown-derive`, закон времени из `logic`, подпись `sign_with_schnorr` побайтово по game-spec §10 с локальной верификацией против дериватного ключа. Состояния нет: ни stable-памяти, ни heap-реестров, ни таймеров, ни `post_upgrade`-миграций.

**DoD.**
- Резолверы двух `subscription_id` различны, 32 байта, стабильны между вызовами — тест в PocketIC.
- `request_release` до срока — отказ; в срок и после — подпись, верифицируемая Ed25519 против резолвера этого `subscription_id` и ни против какого другого. Границы закона — сдвигом времени PocketIC (включительная граница, следующий кусок, переполнение расписания); фазз самого закона — property-тесты G1, канистра зовёт ту же функцию.
- message выпуска побайтово равен `DOMAIN ‖ program_id ‖ escrow ‖ [0x00, index_le]` — пиннуто юнит-тестом на вектор.
- Адрес, выведенный канистрой, равен адресу из `crown-salt` + `crown-derive` на тех же полях — кросс-тест; сами крейты уже паритетны ончейну (фабрика, F4–F5).
- Ретрай `request_release` — эквивалентная валидная подпись того же message, ничего не копится.
- Линт `.did` зелёный; `request_cancel` ещё не существует.

### G3 — Отмена: авторизация донора

**Вход.** game-spec §7, §8.

**Выход.** Update-метод `request_cancel`. Проверка Ed25519-подписи кошелька над сообщением §8 (`"crown:subscription:v1" ‖ lp(chain) ‖ lp(canister_id) ‖ lp(escrow) ‖ ACTION_CANCEL`) против `donor` из предъявленных полей; подпись cancel-message формы тем же путём деривации.

**DoD.**
- Раскладка сообщения §8 пиннута юнит-тестом на вектор; `ACTION_CANCEL = 0` заморожен.
- Чужой ключ, чужой эскроу (другие поля рождения), чужая канистра, чужой кластер — отвергнуты; валидная подпись донора — принята. Матрица негативных тестов в PocketIC.
- Подмена `donor` в полях меняет и адрес, и проверяющий ключ — исполнимого cancel для чужого эскроу не существует; тест.
- message отмены побайтово равен `DOMAIN ‖ program_id ‖ escrow ‖ [0x01]` — пиннуто.
- Повторный запрос с той же подписью — тот же результат.

### G4 — E2e через реальную сеть

**Вход.** game-spec §10.

**Выход.** E2e-скрипт в `scripts/` против devnet: полный цикл подписки через живые фабрику `stream` и сплиттер ядра, с локальным `crown-index`. Драйвер транзакций в `e2e/` (по образцу соседних игр; game-агностичные куски копируются как есть — позиция владельца).

**DoD.**
- **Devnet:** `create_escrow` (короткий `period`, 3+ куска) с резолвером канистры → `request_release` + `release` двух созревших кусков (подпись едет инструкцией `ed25519_program`) → сплиттер эмитит `Settled(payer = эскроу)` на каждый → книга локального `crown-index` показывает донору `+Σ piece` выпущенного → подпись донора → `request_cancel` + `cancel` — остаток вернулся донору. Аномалий ноль.
- `request_release` несозревшего куска — отказ канистры; `release` с подписью созревшего, но не очередного — реверт формы (порядок держит ончейн). Негативные тесты против реального контракта.
- Подпись эскроу другой подписки отвергается реальным контрактом.
- `cancel` и `refund()` не эмитят `Settled`; книга не меняется.
- **`git diff` в `crown-core` и `crown-factory` пуст.** Это и есть настоящий тест шва.

После зелёного G4 игра готова в devnet-объёме: полный цикл «резолвер → эскроу → выпуски → отмена → книга» работает без клиента.

**Выход.**

**DoD.**
- Хэш собранного wasm воспроизводится третьим лицом.

- Первая mainnet-подписка проходит полный цикл, книга показывает донору `+piece` каждого выпуска.
