# Подписка — Стандарты

Правило отбора: **в этом файле только то, что можно проверить машиной или тестом.** Всё, что проверяется вкусом, — не стандарт, а мнение, и здесь ему не место.

## Границы (проверяются CI, не ревью)

| Граница | Как заявлена | Как проверена |
|---|---|---|
| `logic` не делает I/O | ноль зависимостей, время — аргумент | `cargo tree -p subscription-logic` печатает один крейт; `grep -rE 'ic_cdk\|std::(fs\|net\|time)\|reqwest' logic/src/` пусто |
| `logic` не знает про чейн и криптографию | расписание — чистые числа | `grep -riE 'solana\|ed25519\|sha256\|candid' logic/src/` пусто |
| Канистра не двигает деньги | нет переводов, нет знания о сплиттере, флоре, ценах | `grep -riE 'transfer\|approve\|splitter\|fee\|bps\|treasury' canister/src/` пусто |
| Канистра не читает внешние сети | нет outcalls, нет RPC-канистр, нет книги | `grep -r 'http_request' canister/src/` пусто; единственный внешний вызов — management canister |
| Канистра без состояния | нет стабильной памяти, реестров, таймеров | `grep -rE 'stable_structures\|set_timer\|StableBTreeMap' canister/src/` пусто |
| Update-поверхность фиксирована | три метода | парсер `.did` падает на update вне `{get_resolver, request_release, request_cancel}` |
| Авторизация не по principal | только подпись кошелька; выпуск бесправен | `grep -r 'caller' canister/src/` пусто |
| Ядро и фабрика не тронуты | отдельный репозиторий | CI падает, если в репозитории есть путь `crown-core/` или `crown-factory/` |

Зависимости строго в одну сторону: `canister → logic`, `canister → crown-derive` (адрес PDA), `canister → crown-salt` (соль формы `stream` — единственное офчейн-определение, паритет с ончейн `birth_salt` проверяет тест фабрики). Обратных нет. `logic` о существовании канистры не знает.

## Код

**`logic`:**
```rust
#![forbid(unsafe_code)]
#![deny(clippy::unwrap_used, clippy::expect_used, clippy::panic,
        clippy::arithmetic_side_effects, clippy::indexing_slicing)]
```
Только `checked_*` арифметика. Ошибка — значение, не паника. Время — аргумент функции; `logic` не имеет понятия «сейчас».

**`canister`:** `clippy` без предупреждений. `unwrap`/`expect` запрещены на путях ingress; в `#[init]` допустимы — паника там означает несостоявшийся деплой. Каждая выданная threshold-подпись верифицируется локально против дериватного ключа до возврата: подпись, которую цепь отвергла бы, не покидает канистру.

**Криптография:** только пиннутые библиотеки (`ed25519-dalek` или эквивалент). Ничего самописного: ни хешей, ни разбора подписей. Байтовые форматы — сообщение отмены (§8), message формы (§10) — собираются каждый в одном модуле и пиннятся юнит-тестами на конкретные векторы. Соль — только `crown_salt::stream`, никогда руками.

**Общее.** У каждой строки есть текущий потребитель. Ни мёртвого кода, ни обобщённости «на будущее», ни фича-флагов. Сомневаешься между «добавить» и «удалить» — удаляй.

## Тесты, без которых этап не закрывается

1. Расписание: паритет с ончейн-арифметикой, закон, монотонность, детерминизм — property-тестами в `logic`.
2. Деривация: резолверы двух `subscription_id` различны и стабильны; адрес канистры равен `crown-salt` + `crown-derive` на тех же полях; на G4 — равен адресу реально рождённого эскроу.
3. Закон времени: до срока — отказ, после — подпись; фаззом со сдвигом времени PocketIC.
4. Авторизация отмены: чужой ключ, чужой эскроу, чужая канистра, чужой кластер — отвергнуты.
5. Байты подписи: локальный verify против дериватного ключа + принятие реальным контрактом на devnet (e2e: release двух кусков, cancel остатка).
6. Порядок — ончейн: подпись не-очередного созревшего куска реверт формы, не канистры.
7. Ретрай: повторный запрос — тот же результат, ничего не копится.
8. Изоляция: `git diff` в `crown-core` и `crown-factory` пуст.

Тесты лежат рядом с исходником, который проверяют.

## Конфиг

Единая `config/`. Ничего сетевого в коде.

Два профиля: `testnet`, `mainnet`. Смена среды = смена профиля, ноль изменений кода.

Ровно ключи из game-spec §9: `threshold_key`; per-chain — `id`, `factory`, `domain`. Всё. Чейны попарно различны по `(id, factory, domain)` — проверяется при деплое.

Нет и не появится: `crown_index`, RPC-URL, API-ключ, адрес сплиттера, `feeBps`, казна, `release_margin`, `min_gross`, цены, периоды, кворумы, init-оверрайды.

Значения `factory`, `domain` берутся из `crown-factory/deploy/`. Этот репозиторий их не порождает — только копирует.

## Циклы

Канистра комиссий внешней сети **не платит вообще** и ключей для транзакций не держит.

- **Комиссия за `create_escrow` — донор. За `release`/`cancel`/`refund()` — кто нажал.**
- **Циклы канистры — оператор.** Расход: ingress, threshold-деривации и threshold-подписи (по одной на выпуск — у подписки на год их 12+; плюс спам фиктивными полями, game-spec §13.1). Бюджет — G5.
- `get_resolver`, `request_release`, `request_cancel` — бесправные update и потому векторы спама, жгущего циклы. Принято (game-spec §13.1); митигация — rate limit на boundary nodes и бюджет, не код.

## Именование

- Файлы кода: kebab-case. Типы: PascalCase. Функции/переменные: snake_case. Константы: UPPER_SNAKE_CASE.
- Идентификаторы и комментарии — по-английски. Доки и проза — по-русски.
