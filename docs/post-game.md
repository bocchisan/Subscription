# Подписка — Post-game

Всё, чего **нет** в этом репозитории, и почему. Каждая строка — сознательно отложенный кусок с известным швом подключения.

Правило: **пока строка здесь — в репозитории не должно быть ни строчки кода, ни хука, ни поля конфига под неё.** «Подготовить место» — это и есть спекулятивная обобщённость, которую запрещает `standards.md`.

Реализовал — удали строку отсюда.

## Никогда

Не «пока», а **несовместимо с построением**. Изменение любого пункта — новая игра, не правка этой.

| Что | Почему |
|---|---|
| **Состояние в канистре: реестр подписок, эскроу, выданных подписей** | канистра — чистая функция; реестр = write-канал и объект миграции, которых не существует. Порядок держит ончейн-форма |
| **Регистрация / создание подписки** | `subscription_id` непрозрачен, резолвер — деривация. Регистрация дала бы канистре мнение о том, какие подписки «настоящие», — у неё нет и не будет |
| **Проверка существования эскроу в цепи** | канистра не читает внешние сети. Подпись для несуществующего адреса безвредна (game-spec §7) |
| **Знание цен, периодов, каналов** | политика клиента; канистра гарантирует неподделываемость полей, не их осмысленность |
| **Отзыв или срок годности выданной подписи** | нечем помнить; исполнимость гейтит терминальность эскроу (game-spec §13.3) |
| **Отмена кем-либо, кроме донора** | право отмены — у того, чьи деньги заперты. Владелец «закрывает лавочку», просто перестав выпускать: `RELEASE_MARGIN` вернёт остатки донорам |
| **Release раньше срока, пропуск куска, изменение расписания** | расписание нотаризовано адресом эскроу при рождении; закон канистры — зеркало ончейн-закона |
| **Чтение книги, вес, голосование** | в этой игре нет коллективных решений. Нужны — это другая игра |
| **Репутация за что-либо, кроме сеттлмента через сплиттер** | репутация ≡ честно уплаченный доллар. Другой эмиссии нет и не будет |
| **Приз, множитель, ставки** | из эскроу выходит ровно его `GROSS`. Заработать больше внесённого нельзя никому |
| **Своя форма эскроу под эту игру** | `stream` покрывает «время + воля донора» целиком (factory-spec §7). Новая форма — решение `crown-factory`, не игры |
| **Перенос состояния между версиями канистры** | переносить нечего — и это свойство, а не случайность. Новая канистра = новое дерево ключей; старая подписывает свои пути вечно |

## Что сломается первым, и почему это нормально

Канистра — единственный участник игры, который может умереть: кончатся циклы, замолчит подсеть, найдётся баг. Ни один из этих сценариев не теряет деньги: каждый эскроу выходит через `refund()` спустя `RELEASE_MARGIN` после первого невыпущенного срока, без подписи, транзакцией кого угодно. Владелец при этом теряет только несобранные куски — что честно: он их не собрал.

Баг в заблэкхоленной канистре неисправим — как и в ядре, митигация не апгрейд, а замена: новая канистра с новым деревом ключей обслуживает новые подписки, старая подписывает свои пути, пока жива. Репутация не страдает вовсе: она живёт в книге ядра и питается только сеттлментами, которые уже случились.

Поэтому блэкхол дёшев, а G5 спокойно ждёт готовности всей платформы.
